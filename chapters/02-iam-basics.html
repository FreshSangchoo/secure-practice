AWS 계정 생성 과정

AWS를 사용하려면 먼저 계정을 만들어야 합니다. 계정 생성에 필요한 것들을 알아봅시다.

계정 생성에 필요한 것

  • 이메일 주소: 계정의 루트 사용자 이메일이 됩니다 (Gmail, 네이버 등 개인 이메일 가능)
  • 신용카드/체크카드: 프리티어만 사용해도 카드 등록이 필요합니다 (1달러 인증 후 환불됨)
  • 전화번호: 본인 인증용
비유: 은행 계좌 개설과 비슷합니다

AWS 계정을 만드는 것은 은행에서 계좌를 개설하는 것과 비슷합니다. 본인 확인(이메일, 전화번호)을 하고, 결제 수단(카드)을 등록합니다. 프리티어를 사용하면 월 요금은 0원이지만, 혹시 초과 사용 시 요금을 청구하기 위해 카드 등록이 필수입니다.

계정 생성 절차 요약

  1. https://aws.amazon.com 접속 → "무료 계정 생성" 클릭
  2. 이메일 주소 입력 및 계정 이름 설정
  3. 이메일로 받은 인증 코드 입력
  4. 루트 사용자 비밀번호 설정 (매우 강력하게!)
  5. 연락처 정보 입력 (개인/비즈니스 선택)
  6. 결제 정보 입력 (신용카드/체크카드)
  7. 전화번호 인증 (SMS 또는 음성 통화)
  8. 지원 플랜 선택 (무료 Basic 선택)
  9. 계정 생성 완료!
주의: 계정 활성화 시간

계정 생성 후 모든 서비스가 사용 가능해지기까지 최대 24시간이 걸릴 수 있습니다. 실습 전에 미리 계정을 만들어 두세요.

루트 사용자 vs IAM 사용자

AWS 계정을 만들면 자동으로 루트 사용자(Root User)가 생성됩니다. 이 루트 사용자는 계정의 모든 권한을 가지고 있어서 매우 위험합니다.

비유: 건물 마스터키 vs 사원증

루트 사용자 = 건물 마스터키: 모든 방, 금고, 비상구까지 전부 열 수 있습니다. 분실하면 건물 전체가 위험합니다. 평소에는 금고에 보관하고 꼭 필요할 때만 꺼내 씁니다.

IAM 사용자 = 사원증: 자기 사무실과 회의실만 출입할 수 있습니다. 분실해도 해당 사원증만 정지시키면 됩니다. 일상 업무에는 사원증으로 충분합니다.

구분 루트 사용자 IAM 사용자
생성 시점 계정 생성 시 자동 직접 생성
로그인 방법 이메일 + 비밀번호 계정 ID + 사용자명 + 비밀번호
권한 무제한 (계정의 모든 것) 부여된 권한만
사용 용도 계정 설정, 결제 변경 등 특수 상황만 일상적인 AWS 작업
삭제 가능? 불가능 가능
개수 계정당 1개 여러 개 생성 가능
절대 하지 마세요!

루트 사용자로 일상적인 작업을 하는 것은 매우 위험합니다!
루트 계정이 해킹당하면 공격자가 수천만 원어치의 EC2 인스턴스를 생성하거나, 모든 데이터를 삭제할 수 있습니다. 실제로 이런 사고가 자주 발생합니다.

루트 사용자는 MFA를 설정하고, IAM 사용자를 만들어서 그것으로 작업하세요!

MFA란? 왜 필요한가?

MFA(Multi-Factor Authentication, 다중 인증)은 비밀번호 외에 추가적인 인증 수단을 요구하는 보안 방식입니다.

비유: 은행 이체할 때를 생각해보세요

인터넷 뱅킹에서 돈을 보낼 때, 비밀번호만으로는 안 되죠? 비밀번호(1차) + OTP 또는 공인인증서(2차)를 함께 입력해야 합니다. MFA도 똑같은 원리입니다. 비밀번호가 유출되더라도 MFA 코드 없이는 로그인할 수 없어서 계정을 보호할 수 있습니다.

MFA의 인증 요소

  • 1단계 - 알고 있는 것: 비밀번호
  • 2단계 - 가지고 있는 것: 스마트폰의 인증 앱 (Google Authenticator, Microsoft Authenticator 등)

AWS에서 사용할 수 있는 MFA 장치

  • 가상 MFA 장치 (추천): 스마트폰에 Google Authenticator 또는 Authy 앱 설치
  • 보안 키: YubiKey 같은 물리적 보안 키
  • 하드웨어 MFA 장치: 은행 OTP 생성기 같은 전용 장치
추천 MFA 앱

이 강의에서는 Google Authenticator 앱을 사용합니다. 스마트폰 앱 스토어에서 무료로 다운로드할 수 있습니다. (iOS: App Store / Android: Play Store에서 "Google Authenticator" 검색)

IAM 핵심 개념

IAM(Identity and Access Management)은 AWS에서 "누가, 어떤 서비스에, 어떤 작업을 할 수 있는지"를 관리하는 서비스입니다. AWS를 사용한다면 반드시 이해해야 하는 핵심 서비스입니다.

비유: 회사의 인사/보안팀과 같습니다

회사에 입사하면 인사팀이 사원증을 만들어주고, 보안팀이 출입 권한을 설정합니다. 신입 사원은 자기 사무실만 출입 가능하고, 팀장은 회의실도 쓸 수 있고, 임원은 임원 전용 층에도 갈 수 있죠. IAM이 바로 이 역할을 합니다.

IAM의 4가지 핵심 요소

1. 사용자 (User)

AWS를 사용하는 개별 사람 또는 애플리케이션입니다. 각 사용자는 고유한 이름, 비밀번호, Access Key를 가질 수 있습니다.

2. 그룹 (Group)

사용자들의 모음입니다. 같은 권한이 필요한 사용자들을 그룹으로 묶어서 관리하면 편리합니다. 예: "개발팀" 그룹, "운영팀" 그룹, "경영진" 그룹

3. 정책 (Policy)

"무엇을 할 수 있고, 무엇을 할 수 없는지"를 정의한 문서(JSON 형식)입니다. 사용자나 그룹에 정책을 연결(Attach)하면 해당 권한이 부여됩니다.

4. 역할 (Role)

특정 상황에서 임시로 권한을 부여하는 방식입니다. 주로 AWS 서비스 간에 권한을 부여할 때 사용합니다 (예: EC2가 S3에 접근하는 경우).

비유: 호텔에서 생각해보면

사용자 = 호텔 투숙객 (각자 카드키를 받음)
그룹 = 층별 분류 (일반실, 스위트룸, VIP 층)
정책 = 카드키 권한 (자기 방, 수영장, 피트니스 출입 가능 여부)
역할 = 임시 출입증 (회의실을 잠깐 쓸 때 프론트에서 임시 카드 발급)

IAM 정책 이해하기

IAM 정책은 JSON 형식으로 작성됩니다. 처음에는 복잡해 보이지만, 구조를 이해하면 읽을 수 있습니다.

정책의 기본 구조

json 
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::my-bucket/*"
    }
  ]
}
  • Effect: 허용(Allow) 또는 거부(Deny)
  • Action: 어떤 작업? (예: s3:GetObject = S3에서 파일 읽기)
  • Resource: 어떤 리소스에 대해? (특정 S3 버킷, 특정 EC2 등)
비유: 통행증으로 읽어보면

위 정책을 사람 말로 번역하면:
"이 사용자는 my-bucket이라는 S3 버킷에서 파일을 읽는 것을 허용합니다."

마치 "이 사원증으로 3층 문서 보관실에서 문서를 열람할 수 있습니다" 같은 것이죠.

AWS 관리형 정책 (자주 사용하는 것)

정책 이름 설명
AdministratorAccess 모든 서비스에 대한 전체 권한 (루트와 거의 동일)
AmazonEC2FullAccess EC2 서비스에 대한 전체 권한
AmazonS3ReadOnlyAccess S3 읽기만 가능
AmazonVPCFullAccess VPC 서비스에 대한 전체 권한
최소 권한 원칙 (Principle of Least Privilege)

IAM에서 가장 중요한 원칙은 "필요한 최소한의 권한만 부여하라"입니다. 모든 사용자에게 AdministratorAccess를 주면 편하지만, 보안 사고 시 피해가 커집니다. 이 강의에서는 학습 편의를 위해 AdministratorAccess를 사용하지만, 실무에서는 꼭 필요한 권한만 부여해야 합니다.

IAM 역할 (Role)

IAM 역할은 사람이 아닌 AWS 서비스에게 권한을 부여할 때 주로 사용합니다.

비유: 대리 운전 기사

내 차를 대리 기사에게 맡기면, 기사에게 임시로 내 차를 운전할 권한을 줍니다. 목적지에 도착하면 권한은 사라집니다.

마찬가지로, EC2 서버가 S3에 파일을 업로드해야 할 때, EC2에 "S3 접근 역할"을 부여합니다. EC2는 이 역할을 통해 임시 자격 증명을 받아 S3에 접근합니다.

역할이 필요한 대표적인 상황:

  • EC2 → S3: EC2 서버에서 S3에 파일 업로드/다운로드
  • Lambda → DynamoDB: Lambda 함수에서 DynamoDB 테이블 읽기/쓰기
  • 다른 AWS 계정의 사용자가 내 리소스에 접근해야 할 때
역할은 나중에 자세히!

IAM 역할은 Chapter 05(EC2 배포)부터 실제로 사용하게 됩니다. 지금은 "AWS 서비스에게 권한을 줄 때 역할을 사용한다"는 것만 기억해두세요.

실습

실습 1: AWS 계정 생성
  1. AWS 홈페이지 접속

    웹 브라우저에서 https://aws.amazon.com에 접속합니다.

    오른쪽 상단의 "AWS 계정 생성" 또는 "Create an AWS Account" 버튼을 클릭합니다.

    [스크린샷: AWS 홈페이지 - 계정 생성 버튼 위치]
  2. 이메일 주소 및 계정 이름 입력

    Root user email address: 본인의 이메일 주소를 입력합니다.

    AWS account name: 계정 이름을 입력합니다 (예: "my-aws-study").

    "Verify email address" 버튼을 클릭하면 인증 코드가 이메일로 발송됩니다.

    [스크린샷: 이메일, 계정 이름 입력 화면]
  3. 인증 코드 입력 및 비밀번호 설정

    이메일로 받은 6자리 인증 코드를 입력합니다.

    루트 사용자 비밀번호를 설정합니다. 반드시 강력한 비밀번호를 사용하세요!

    • 8자 이상
    • 대문자, 소문자, 숫자, 특수문자 포함
    • 다른 사이트와 다른 고유한 비밀번호
  4. 연락처 정보 입력

    How do you plan to use AWS? → "Personal" 선택 (개인 학습용)

    이름, 전화번호, 주소 등을 입력합니다.

  5. 결제 정보 입력

    신용카드 또는 체크카드 정보를 입력합니다.

    인증을 위해 $1 (약 1,300원)이 일시적으로 결제되었다가 환불됩니다.

    카드 등록 필수

    프리티어만 사용할 예정이라도 카드 등록은 필수입니다. 프리티어 범위 내에서만 사용하면 요금이 나오지 않습니다.

  6. 전화번호 인증

    전화번호를 입력하고 보안 코드를 받아 입력합니다.

  7. 지원 플랜 선택

    "Basic support - Free"를 선택합니다.

    "Complete sign up" 버튼을 클릭하면 계정 생성이 완료됩니다!

실습 2: 루트 계정 MFA 설정
  1. AWS 콘솔 로그인

    https://console.aws.amazon.com에 접속합니다.

    "Root user"를 선택하고, 계정 생성 시 사용한 이메일과 비밀번호로 로그인합니다.

    [스크린샷: AWS 콘솔 로그인 화면 - Root user 선택]
  2. IAM 대시보드 이동

    상단 검색바에 "IAM"을 검색하고 클릭합니다.

    또는 서비스 메뉴 → 보안, 자격 증명 및 규정 준수 → IAM을 클릭합니다.

    [스크린샷: IAM 검색 및 대시보드 화면]
  3. 루트 사용자에 MFA 추가

    IAM 대시보드에서 "보안 권장 사항" 또는 "Security recommendations" 영역을 찾습니다.

    "Add MFA for root user" 또는 "루트 사용자에 MFA 추가"를 클릭합니다.

    "MFA 활성화" 버튼을 클릭합니다.

  4. MFA 디바이스 선택

    Device name: 알아보기 쉬운 이름 입력 (예: "my-phone")

    MFA device: "Authenticator app" 선택

    "Next" 클릭

  5. QR 코드 스캔

    스마트폰에서 Google Authenticator 앱을 엽니다.

    앱에서 "+" 버튼 → "QR 코드 스캔"을 선택합니다.

    화면에 표시된 QR 코드를 스마트폰 카메라로 스캔합니다.

    [스크린샷: QR 코드가 표시된 MFA 설정 화면]
  6. MFA 코드 입력

    Google Authenticator 앱에 나타나는 6자리 숫자를 "MFA code 1"에 입력합니다.

    30초 후 새로운 6자리 숫자가 나타나면 그것을 "MFA code 2"에 입력합니다.

    "Add MFA" 버튼을 클릭하면 MFA 설정이 완료됩니다!

실습 3: IAM 사용자 만들기
  1. IAM 사용자 페이지 이동

    IAM 콘솔 왼쪽 메뉴에서 "사용자(Users)"를 클릭합니다.

    오른쪽 상단의 "사용자 생성(Create user)" 버튼을 클릭합니다.

    [스크린샷: IAM 사용자 목록 화면 - 사용자 생성 버튼]
  2. 사용자 정보 입력

    User name: 사용자 이름을 입력합니다 (예: admin-user)

    "Provide user access to the AWS Management Console"에 체크합니다.

    "I want to create an IAM user"를 선택합니다.

    Console password: "Custom password"를 선택하고 비밀번호를 입력합니다.

    "Users must create a new password at next sign-in" 체크 해제 (실습 편의상)

    "Next" 클릭

  3. 권한 설정

    "Attach policies directly"를 선택합니다.

    검색창에 "AdministratorAccess"를 입력합니다.

    AdministratorAccess 정책에 체크합니다.

    "Next" 클릭

    [스크린샷: 정책 연결 화면 - AdministratorAccess 선택]
