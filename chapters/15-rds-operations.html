RDS 운영

학습 목표

  • RDS 자동 백업과 수동 스냅샷의 차이를 이해한다
  • 수동 스냅샷을 생성하고 복원할 수 있다
  • 파라미터 그룹의 역할을 이해한다
  • RDS 운영에 필요한 기본 작업을 수행할 수 있다

자동 백업

RDS는 매일 자동으로 데이터베이스 전체를 백업합니다. 이 백업을 사용하면 특정 시점으로 데이터를 복원할 수 있습니다.

비유로 이해하기: 블랙박스 녹화

자동차의 블랙박스가 계속 녹화하다가 일정 기간이 지나면 자동으로 오래된 영상을 삭제하는 것과 같습니다. RDS 자동 백업도 매일 DB 상태를 "녹화"하고, 설정된 보존 기간이 지나면 자동 삭제됩니다. 사고(데이터 손실)가 발생하면 원하는 시점의 "녹화 영상"을 재생(복원)할 수 있습니다.

자동 백업의 특징

항목설명
백업 주기매일 1회 (백업 윈도우 시간에)
보존 기간1~35일 (기본 7일)
복원 방식특정 시점 복원 (Point-in-Time Recovery)
트랜잭션 로그5분 간격으로 S3에 저장
비용DB 크기만큼의 백업 스토리지 무료
삭제 시RDS 인스턴스 삭제 시 자동 백업도 삭제
특정 시점 복원 (Point-in-Time Recovery)

자동 백업 + 트랜잭션 로그를 조합하면 최근 5분 전까지의 어떤 시점으로든 복원할 수 있습니다. 예를 들어 "오늘 오후 2시 30분 시점"으로 DB를 복원하는 것이 가능합니다. 단, 복원하면 기존 인스턴스가 수정되는 것이 아니라 새로운 RDS 인스턴스가 생성됩니다.

백업 윈도우

자동 백업이 실행되는 시간대입니다. 백업 중에는 I/O가 일시적으로 지연될 수 있으므로, 트래픽이 적은 시간대(예: 새벽 3시~4시)로 설정합니다.

Multi-AZ에서는 영향 없음

다중 AZ를 사용하면 Standby에서 백업이 실행되므로 Primary의 성능에 영향이 없습니다. 단일 AZ에서는 백업 시 잠깐의 I/O 지연이 발생할 수 있습니다.

수동 스냅샷

자동 백업과 별도로, 원하는 시점에 수동으로 스냅샷을 생성할 수 있습니다.

비유로 이해하기: 사진 찍기

자동 백업이 CCTV(자동 녹화)라면, 수동 스냅샷은 사진 찍기입니다. 중요한 순간(배포 전, 대규모 데이터 변경 전)에 "찰칵!" 하고 사진을 찍어두면, 문제가 생겼을 때 그 사진 시점으로 돌아갈 수 있습니다.

자동 백업 vs 수동 스냅샷

구분자동 백업수동 스냅샷
생성AWS가 매일 자동사용자가 원할 때 수동
보존1~35일 후 자동 삭제사용자가 삭제할 때까지 영구 보관
RDS 삭제 시함께 삭제됨유지됨 (영구)
용도일상적인 복구배포 전 백업, 마이그레이션
비용DB 크기만큼 무료초과분 유료 ($0.095/GB/월)
수동 스냅샷 활용 시점

다음과 같은 상황에서 수동 스냅샷을 생성하세요:

  • 대규모 스키마 변경(ALTER TABLE) 전
  • 새 버전 배포 전
  • 대량의 데이터 삭제/수정 전
  • RDS 인스턴스를 삭제하기 전 (데이터 보존용)

스냅샷에서 복원

스냅샷에서 복원하면 새로운 RDS 인스턴스가 생성됩니다. 기존 인스턴스가 수정되는 것이 아닙니다.

비유로 이해하기

스냅샷 복원은 사진으로 새 그림을 그리는 것과 같습니다. 원본 그림(기존 RDS)은 그대로 있고, 사진(스냅샷)을 보고 동일한 새 그림(새 RDS)을 그립니다. 새 그림이 완성되면 원본 대신 사용할 수 있습니다.

복원 시 주의사항
  • 복원된 인스턴스는 새 엔드포인트를 가집니다 (기존 엔드포인트와 다름)
  • 보안그룹은 기본(default)으로 초기화됩니다 → 수동으로 sg-rds 설정 필요
  • 파라미터 그룹도 재설정이 필요할 수 있습니다
  • 복원에는 10~30분이 소요될 수 있습니다

파라미터 그룹

파라미터 그룹은 MySQL의 설정 파일(my.cnf)을 AWS 방식으로 관리하는 것입니다.

비유로 이해하기

파라미터 그룹은 자동차의 세팅 프로필과 같습니다. "시트 위치, 미러 각도, 에어컨 온도"를 프로필로 저장해두면 운전자에 맞게 바로 적용할 수 있듯이, DB의 "메모리 할당, 문자셋, 타임존" 등을 프로필로 관리합니다.

주요 파라미터

파라미터설명권장값
character_set_server서버 문자셋utf8mb4
collation_server정렬 규칙utf8mb4_unicode_ci
time_zone타임존Asia/Seoul
max_connections최대 동시 연결 수인스턴스에 따라 자동
slow_query_log느린 쿼리 로그1 (활성화)
한국어와 이모지를 위한 utf8mb4

MySQL에서 한국어와 이모지를 정상적으로 처리하려면 문자셋을 utf8mb4로 설정해야 합니다. 기본 파라미터 그룹은 수정할 수 없으므로, 커스텀 파라미터 그룹을 만들어 적용합니다.

실습

실습: RDS 백업 및 복원
  1. 자동 백업 설정 확인/변경

    AWS 콘솔 → RDS → Databases → shopeasy-db 클릭

    Maintenance & backups 탭에서 현재 설정을 확인합니다:

    • Automated backups: Enabled (활성화 상태)
    • Backup retention period: 7 days
    • Backup window: 기본값 (AWS 자동 설정)
    • Latest restore time: 최근 복원 가능 시점
    [스크린샷: RDS 백업 설정 확인]

    백업 윈도우를 변경하고 싶다면:

    Modify 클릭 → Backup 섹션에서:

    • Backup retention period: 7 days (변경 가능)
    • Backup window: Select window에서 18:00-18:30 UTC (한국시간 새벽 3시~3시 30분)

    변경 시 Apply immediately 또는 Apply during the next scheduled maintenance window 선택

    UTC 시간 변환

    AWS는 UTC 시간을 사용합니다. 한국시간(KST)은 UTC+9이므로:
    UTC 18:00 = KST 03:00 (새벽 3시)
    트래픽이 적은 새벽 시간대로 설정하는 것이 좋습니다.

  2. 현재 데이터 확인 (복원 후 비교용)

    스냅샷 생성 전에 현재 데이터를 확인해둡니다.

    bash
    # API 서버 EC2에서 RDS 접속
mysql -h shopeasy-db.xxxxx.ap-northeast-2.rds.amazonaws.com -u admin -p
    sql
    USE shopeasy;

-- 현재 상품 개수 확인
SELECT COUNT(*) AS total_products FROM products;

-- 상품 목록 확인
SELECT id, name, price FROM products ORDER BY id;

-- 결과를 메모해두세요 (복원 후 비교용)

    테스트를 위해 데이터를 하나 추가합니다:

    sql
    -- 테스트 상품 추가 (스냅샷 이후 데이터)
INSERT INTO products (name, price, description, image)
VALUES ('테스트 상품 - 스냅샷 이후', 99999, '이 상품은 스냅샷 이후에 추가됨', '/images/test.jpg');

-- 확인
SELECT id, name, price FROM products ORDER BY id DESC LIMIT 3;

EXIT;
  3. 수동 스냅샷 생성

    AWS 콘솔 → RDS → Databases → shopeasy-db 선택

    Actions → Take snapshot

    • DB snapshot identifier: shopeasy-db-manual-backup-01

    Take snapshot 클릭

    스냅샷 생성 상태를 확인합니다:

    RDS → Snapshots에서 Status가 Creating → Available이 될 때까지 기다립니다 (3~5분).

    [스크린샷: 수동 스냅샷 생성 - Available 상태]
    스냅샷의 내용

    이 스냅샷에는 "테스트 상품 - 스냅샷 이후" 데이터가 포함되어 있지 않습니다. 스냅샷은 "Take snapshot"을 클릭한 시점의 데이터를 담고 있습니다. (실제로는 스냅샷 시작 시점이므로, 방금 추가한 데이터가 포함될 수도 있습니다. 정확한 시점 복원이 필요하면 Point-in-Time Recovery를 사용하세요.)

  4. 스냅샷에서 새 RDS 인스턴스 복원

    AWS 콘솔 → RDS → Snapshots → shopeasy-db-manual-backup-01 선택

    Actions → Restore snapshot

    Settings

    • DB instance identifier: shopeasy-db-restored

    Instance configuration

    • DB instance class: db.t3.micro

    Connectivity

    • VPC: my-vpc
    • DB subnet group: dbsubnet-shopeasy
    • Public access: No

    Restore DB instance 클릭

    복원에 10~15분 소요됩니다

    Status가 Available이 될 때까지 기다리세요.

    비용 주의

    복원된 RDS 인스턴스도 별도의 비용이 발생합니다. 프리티어는 RDS 1대만 무료이므로, 확인이 끝나면 복원된 인스턴스를 삭제하세요.

  5. 복원된 인스턴스 보안그룹 설정

    복원된 인스턴스는 기본 보안그룹(default)이 적용되어 있습니다. sg-rds로 변경합니다.

    RDS → Databases → shopeasy-db-restored → Modify

    • Security group: sg-rds 선택, default 제거

    Continue → Apply immediately → Modify DB instance

  6. 복원된 인스턴스 데이터 확인

    복원된 인스턴스의 엔드포인트를 확인합니다:

    RDS → Databases → shopeasy-db-restored → Connectivity & security

    새 엔드포인트: shopeasy-db-restored.xxxxx.ap-northeast-2.rds.amazonaws.com

    bash
    # 복원된 RDS에 접속
mysql -h shopeasy-db-restored.xxxxx.ap-northeast-2.rds.amazonaws.com -u admin -p
    sql
    USE shopeasy;

-- 상품 데이터 확인
SELECT id, name, price FROM products ORDER BY id;

-- "테스트 상품 - 스냅샷 이후" 데이터 존재 여부 확인
SELECT * FROM products WHERE name LIKE '%테스트%';

-- 스냅샷 시점의 데이터만 존재하는 것을 확인!

EXIT;
    [스크린샷: 복원된 RDS에서 데이터 확인]
    복원 결과 확인

    스냅샷 시점의 데이터가 복원되었습니다. 스냅샷 이후에 추가한 "테스트 상품"은 스냅샷에 포함되지 않았으므로 복원된 인스턴스에는 없습니다. 이것이 바로 스냅샷/백업의 핵심입니다: 원하는 시점으로 되돌리기.

  7. 실습 정리 - 복원된 인스턴스 삭제

    비용 절약을 위해 복원된 인스턴스를 삭제합니다.

    RDS → Databases → shopeasy-db-restored → Actions → Delete

    • Create final snapshot: 체크 해제 (학습용이므로)
    • I acknowledge...: 체크
    • delete me 입력

    Delete 클릭

    원본 RDS(shopeasy-db)에서 테스트 데이터도 정리합니다:

    sql
    -- 원본 RDS에 접속 후
USE shopeasy;
DELETE FROM products WHERE name LIKE '%테스트%';

-- 확인
SELECT COUNT(*) FROM products;
  8. (선택) 커스텀 파라미터 그룹 생성

    한국어와 이모지 지원을 위해 utf8mb4 파라미터 그룹을 만듭니다.

    AWS 콘솔 → RDS → Parameter groups → Create parameter group

    • Parameter group family: mysql8.0
    • Group Name: shopeasy-mysql-params
    • Description: ShopEasy MySQL parameters - utf8mb4, Asia/Seoul

    Create 클릭

    파라미터를 수정합니다:

    shopeasy-mysql-params 클릭 → Edit parameters

    검색하여 다음 파라미터를 변경:

    파라미터값
    character_set_clientutf8mb4
    character_set_connectionutf8mb4
    character_set_databaseutf8mb4
    character_set_resultsutf8mb4
    character_set_serverutf8mb4
    collation_connectionutf8mb4_unicode_ci
    collation_serverutf8mb4_unicode_ci
    time_zoneAsia/Seoul

    Save changes 클릭

    RDS 인스턴스에 파라미터 그룹을 적용합니다:

    RDS → Databases → shopeasy-db → Modify

    • DB parameter group: shopeasy-mysql-params

    Continue → Apply immediately → Modify DB instance

    파라미터 그룹 적용 후 재부팅 필요

    일부 파라미터는 RDS 재부팅 후 적용됩니다. RDS → shopeasy-db → Actions → Reboot으로 재부팅하세요. 재부팅 중에는 잠시 서비스가 중단됩니다 (약 1~2분).

    [스크린샷: 파라미터 그룹 설정 화면]
