AWS 관리 콘솔 둘러보기

AWS 관리 콘솔(Management Console)은 웹 브라우저에서 AWS 서비스를 관리하는 그래픽 인터페이스(GUI)입니다. 마우스 클릭만으로 서버를 만들고, 데이터베이스를 설정하고, 네트워크를 구성할 수 있습니다.

비유: 스마트폰의 설정 앱

스마트폰에서 Wi-Fi 연결, 화면 밝기 조절, 앱 권한 관리를 "설정" 앱에서 하듯이, AWS 서비스의 모든 관리를 "관리 콘솔"에서 합니다. 복잡한 명령어를 외울 필요 없이, 화면을 보면서 클릭으로 설정할 수 있습니다.

콘솔 화면의 주요 구성 요소

1. 상단 내비게이션 바

  • 서비스 메뉴: 모든 AWS 서비스를 카테고리별로 탐색
  • 검색바: 서비스 이름을 직접 검색 (가장 빠른 방법!)
  • 리전 선택: 현재 작업 중인 리전 표시 및 변경
  • 계정 메뉴: 계정 설정, 로그아웃 등
  • CloudShell: 브라우저에서 바로 CLI를 사용할 수 있는 터미널
[스크린샷: AWS 콘솔 상단 내비게이션 바 - 각 요소 표시]

2. 최근 방문 서비스

콘솔 홈 화면에는 최근에 사용한 서비스 목록이 표시됩니다. 자주 사용하는 서비스는 즐겨찾기(별표)에 추가할 수 있습니다.

3. 서비스별 대시보드

각 서비스에 진입하면 해당 서비스의 대시보드가 나옵니다. 왼쪽에 세부 메뉴, 가운데에 리소스 목록이 표시됩니다.

콘솔 사용 팁

서비스를 찾을 때는 상단의 검색바를 사용하는 것이 가장 빠릅니다. "EC2"를 타이핑하면 바로 EC2 서비스로 이동할 수 있습니다. 자주 사용하는 서비스는 별표(즐겨찾기)를 눌러두면 상단 바에 고정됩니다.

리전 선택의 중요성

AWS에서 리소스를 만들 때 반드시 리전을 확인해야 합니다. 리전에 따라 리소스의 위치가 결정되고, 다른 리전에 만든 리소스는 현재 리전에서 보이지 않습니다.

비유: 네이버 카페의 게시판

네이버 카페에서 "서울 모임" 게시판에 글을 쓰면 "부산 모임" 게시판에서는 보이지 않죠? AWS도 마찬가지입니다. 서울 리전에 만든 EC2 서버는 도쿄 리전 화면에서는 보이지 않습니다. "분명히 서버를 만들었는데 안 보여요!"라는 문제의 90%는 리전을 잘못 선택한 것입니다.

가장 흔한 실수!

초보자가 가장 많이 하는 실수가 리전을 확인하지 않는 것입니다.
- EC2를 서울에 만들었는데, VPC를 도쿄에 만들면 연결이 안 됩니다
- 서울에서 리소스를 만들었는데 도쿄로 바뀌어 있으면 "내 리소스가 없어졌어요!"
- 여러 리전에 리소스를 만들면 프리티어가 더 빨리 소진됩니다

실습을 시작하기 전에 항상 오른쪽 상단의 리전이 "서울(ap-northeast-2)"인지 확인하세요!

서울 리전 (ap-northeast-2)

이 강의의 모든 실습은 서울 리전(ap-northeast-2)에서 진행합니다. 콘솔 오른쪽 상단에서 현재 리전을 확인하고, 서울이 아니라면 변경하세요.

[스크린샷: 콘솔 오른쪽 상단 리전 선택 드롭다운 - 서울 리전 선택]

글로벌 서비스 vs 리전 서비스

대부분의 서비스는 리전별로 동작하지만, 일부 서비스는 리전과 무관하게 글로벌로 동작합니다.

글로벌 서비스 리전 서비스
IAM (사용자/권한) EC2 (가상 서버)
Route 53 (DNS) VPC (네트워크)
CloudFront (CDN) S3 (스토리지)*
AWS Organizations RDS (데이터베이스)

* S3 버킷은 특정 리전에 생성되지만, 이름은 전 세계적으로 고유해야 합니다.

AWS CLI란? 왜 사용하나?

AWS CLI(Command Line Interface)는 터미널(명령 프롬프트)에서 텍스트 명령어를 입력하여 AWS 서비스를 관리하는 도구입니다.

비유: 관리 콘솔 = 리모컨, CLI = 음성 명령

TV를 볼 때 리모컨(관리 콘솔)으로 채널을 바꿀 수도 있고, "OK Google, TV 채널 7번으로 바꿔줘"라고 말(CLI)할 수도 있습니다. 결과는 같지만, 음성 명령은 더 빠르고 자동화하기 쉽습니다.

AWS CLI도 마찬가지입니다. 콘솔에서 10번 클릭해야 할 작업을 CLI 명령어 한 줄로 끝낼 수 있습니다.

CLI를 사용하는 이유

  • 자동화: 스크립트로 반복 작업을 자동화할 수 있습니다
  • 속도: 콘솔에서 여러 화면을 거쳐야 할 작업을 명령어 한 줄로
  • 일괄 처리: 수십 개의 리소스를 한 번에 생성/삭제
  • 원격 접속: SSH로 접속한 서버에서 AWS 서비스 관리
  • CI/CD 통합: 배포 파이프라인에서 AWS 자원 관리

AWS 접근 방법 비교

방법 특징 적합한 상황
관리 콘솔 웹 브라우저 GUI 처음 배울 때, 시각적 확인이 필요할 때
AWS CLI 터미널 명령어 반복 작업, 자동화, 빠른 조회
AWS SDK 프로그래밍 언어별 라이브러리 애플리케이션에서 AWS 연동
CloudFormation/CDK 인프라를 코드로 정의 인프라 자동 구축, 재현 가능한 환경
이 강의에서는?

이 강의에서는 관리 콘솔을 메인으로 사용하면서, 필요할 때 CLI 명령어도 함께 소개합니다. 콘솔에서 개념을 시각적으로 이해한 후, CLI로 빠르게 같은 작업을 해보는 방식입니다.

실습

실습 1: AWS CLI 설치
  1. 운영체제 확인

    본인의 운영체제에 맞는 설치 방법을 따르세요.

  2. Windows에 설치

    아래 링크에서 MSI 설치 파일을 다운로드하여 실행합니다.

    다운로드: https://awscli.amazonaws.com/AWSCLIV2.msi

    설치 마법사의 안내에 따라 "Next"를 클릭하여 설치를 완료합니다.

    설치 후 명령 프롬프트(CMD) 또는 PowerShell을 새로 열어 확인합니다:

    cmd 
    aws --version

    결과 예시: aws-cli/2.x.x Python/3.x.x Windows/10 exe/AMD64

  3. macOS에 설치

    터미널에서 다음 명령어를 실행합니다:

    bash 
    curl "https://awscli.amazonaws.com/AWSCLIV2.pkg" -o "AWSCLIV2.pkg"
sudo installer -pkg AWSCLIV2.pkg -target /

    또는 Homebrew가 설치되어 있다면:

    bash 
    brew install awscli

    설치 확인:

    bash 
    aws --version
  4. Linux에 설치
    bash 
    curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o "awscliv2.zip"
unzip awscliv2.zip
sudo ./aws/install

    설치 확인:

    bash 
    aws --version
실습 2: Access Key 생성 및 aws configure 설정
  1. IAM 사용자의 Access Key 생성

    AWS 콘솔에 IAM 사용자로 로그인합니다. (Chapter 02에서 만든 admin-user)

    오른쪽 상단의 사용자 이름을 클릭 → "보안 자격 증명(Security credentials)"을 클릭합니다.

    [스크린샷: 사용자 메뉴 - Security credentials 위치]
  2. Access Key 만들기

    "액세스 키(Access keys)" 섹션으로 스크롤합니다.

    "액세스 키 만들기(Create access key)"를 클릭합니다.

    사용 사례에서 "Command Line Interface (CLI)"를 선택합니다.

    하단의 확인 체크박스에 체크하고 "Next"를 클릭합니다.

    설명 태그는 비워두고 "Create access key"를 클릭합니다.

  3. Access Key 저장

    생성된 Access Key ID와 Secret Access Key를 반드시 안전한 곳에 저장합니다!

    Secret Access Key는 이 화면에서만!

    Secret Access Key는 생성 직후 이 화면에서만 볼 수 있습니다. 페이지를 닫으면 다시 볼 수 없습니다! 반드시 복사하거나 CSV 파일을 다운로드하세요. 또한, 이 키를 절대로 GitHub, 블로그 등에 공개하지 마세요. 공개된 키로 수천만 원의 요금이 발생한 사례가 실제로 있습니다.

    [스크린샷: Access Key 생성 완료 화면 - Key ID와 Secret Key 표시]
  4. aws configure 실행

    터미널(CMD, PowerShell, 또는 Terminal)을 열고 다음 명령어를 실행합니다:

    bash 
    aws configure

    다음 4가지를 차례로 입력합니다:

    text 
    AWS Access Key ID [None]: (여기에 Access Key ID 입력)
AWS Secret Access Key [None]: (여기에 Secret Access Key 입력)
Default region name [None]: ap-northeast-2
Default output format [None]: json
    설정 의미

    Default region: CLI에서 리전을 지정하지 않으면 서울 리전을 기본으로 사용합니다.
    Default output format: 결과를 JSON 형식으로 보여줍니다. (json, text, table 중 선택)

  5. 설정 확인

    설정이 제대로 되었는지 확인합니다:

    bash 
    aws sts get-caller-identity

    정상적으로 설정되었다면 다음과 비슷한 결과가 나옵니다:

    json 
    {
    "UserId": "AIDA...............",
    "Account": "123456789012",
    "Arn": "arn:aws:iam::123456789012:user/admin-user"
}

    본인의 계정 ID와 IAM 사용자 이름이 표시되면 성공입니다!

실습 3: 기본 CLI 명령어 실행
  1. S3 버킷 목록 조회

    현재 계정의 S3 버킷 목록을 조회합니다 (아직 버킷이 없으면 아무것도 안 나옵니다):

    bash 
    aws s3 ls
  2. EC2 인스턴스 목록 조회

    현재 리전의 EC2 인스턴스 목록을 조회합니다 (아직 없으면 빈 목록):

    bash 
    aws ec2 describe-instances

    테이블 형식으로 보고 싶다면:

    bash 
    aws ec2 describe-instances --output table
  3. 현재 리전 확인
    bash 
    aws configure get region

    결과: ap-northeast-2 (서울 리전)

  4. IAM 사용자 목록 조회
    bash 
    aws iam list-users

    Chapter 02에서 만든 admin-user가 목록에 보이면 정상입니다.

  5. 사용 가능한 리전 목록 조회
    bash 
    aws ec2 describe-regions --output table

    전 세계 AWS 리전 목록이 표시됩니다.

CLI 명령어 구조

AWS CLI 명령어는 항상 이런 구조입니다:

aws [서비스] [명령] [옵션]

예시:
aws s3 ls → S3 서비스의 ls(목록 보기) 명령
aws ec2 describe-instances → EC2 서비스의 인스턴스 목록 조회
aws iam list-users → IAM 서비스의 사용자 목록 조회

도움말이 필요하면 aws [서비스] help를 입력하세요.
