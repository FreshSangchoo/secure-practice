이커머스 서비스 소개

학습 목표

  • 실습용 ShopEasy 쇼핑몰의 구조와 기능을 이해할 수 있다
  • 프론트엔드(React)와 백엔드(Node.js) 의 역할을 구분할 수 있다
  • 최종 AWS 아키텍처의 전체 그림을 미리 파악할 수 있다
  • 로컬 환경에서 쇼핑몰 서비스를 실행할 수 있다

ShopEasy 쇼핑몰 소개

이 강의에서는 ShopEasy라는 간단한 쇼핑몰 서비스를 AWS에 배포하면서 각종 클라우드 서비스를 배웁니다. 처음에는 로컬 컴퓨터에서 실행하고, 강의가 진행되면서 하나씩 AWS 서비스로 옮겨갑니다.

비유: 레고 블록 쌓기

ShopEasy를 AWS에 배포하는 과정은 레고 블록을 하나씩 쌓아가는 것과 같습니다. 처음에는 바닥판(로컬 실행)에서 시작해서, 벽돌(EC2), 지붕(VPC), 창문(RDS), 페인트(S3), 간판(CloudFront)을 하나씩 추가하여 완성된 집을 만듭니다. 각 챕터가 하나의 레고 블록이라고 생각하세요!

왜 이커머스 서비스를 실습 소재로?

  • 실무에 가까움: 대부분의 웹 서비스가 이커머스와 유사한 구조를 가집니다
  • 다양한 AWS 서비스 활용: 서버, DB, 파일 저장, CDN, 서버리스 등 거의 모든 주요 서비스를 사용합니다
  • 이해하기 쉬움: 누구나 온라인 쇼핑 경험이 있으므로 기능을 직관적으로 이해할 수 있습니다

프로젝트 구조

ShopEasy는 프론트엔드와 API 서버로 나뉘어 있습니다.

비유: 레스토랑으로 이해하기

프론트엔드 = 레스토랑 홀 (손님이 보는 부분): 메뉴판, 인테리어, 테이블 배치 등 손님이 직접 보고 상호작용하는 공간입니다.

API 서버 = 주방 (뒤에서 일하는 부분): 주문을 받고, 요리를 하고, 재료를 관리하는 곳입니다. 손님은 주방에 직접 가지 않고, 웨이터(API)를 통해 주문합니다.

데이터베이스 = 식재료 창고: 모든 재료(데이터)가 보관되는 곳입니다.

text 
ecommerce-app/
├── frontend/               # 프론트엔드 (React)
│   ├── public/             # 정적 파일
│   ├── src/
│   │   ├── components/     # 재사용 가능한 UI 컴포넌트
│   │   ├── pages/          # 페이지 컴포넌트
│   │   ├── context/        # 전역 상태 관리 (장바구니 등)
│   │   ├── api.js          # API 서버 통신 모듈
│   │   ├── App.jsx         # 메인 앱 컴포넌트
│   │   └── main.jsx        # 진입점
│   └── package.json
│
└── api-server/             # API 서버 (Node.js + Express)
    ├── src/
    │   ├── app.js          # Express 앱 설정
    │   ├── config/         # 환경 설정
    │   ├── routes/         # API 라우트 (상품, 리뷰, 장바구니 등)
    │   ├── services/       # 비즈니스 로직
    │   └── middleware/     # 미들웨어 (에러 처리 등)
    ├── data/               # SQLite DB 파일 (로컬용)
    ├── uploads/            # 업로드된 파일 저장 (로컬용)
    ├── seed.js             # 테스트 데이터 생성 스크립트
    └── package.json

API란?

API(Application Programming Interface)는 프로그램들이 서로 대화하는 방법입니다. 프론트엔드가 "상품 목록을 주세요"라고 요청(Request)하면, API 서버가 데이터베이스에서 데이터를 가져와 JSON 형태로 응답(Response)합니다.

예시: GET /api/products → 상품 목록 반환
예시: POST /api/cart → 장바구니에 상품 추가

주요 기능

ShopEasy는 다음과 같은 기능을 제공합니다. 각 기능이 어떤 AWS 서비스와 연결되는지 미리 살펴봅시다.

기능 설명 관련 AWS 서비스
상품 목록 전체 상품을 카드 형태로 표시 EC2 (API), RDS (데이터)
상품 상세 개별 상품의 정보, 가격, 설명 표시 EC2, RDS
리뷰 (글+사진) 상품에 대한 텍스트 리뷰와 사진 리뷰 작성 DynamoDB (리뷰 데이터), S3 (리뷰 사진)
장바구니 상품을 장바구니에 담고 수량 조절 EC2, RDS
가짜 구매 결제 시뮬레이션 (실제 결제 없음) SQS (주문 처리), SNS (알림)
처음에는 간단하게, 나중에 확장

처음(Chapter 05~06)에는 모든 것을 EC2 하나에서 실행합니다. 강의가 진행되면서 데이터베이스를 RDS로, 파일 저장을 S3로, 프론트엔드를 CloudFront로 하나씩 분리해 나갑니다. 이 과정에서 자연스럽게 "왜 이 서비스가 필요한가?"를 체감하게 됩니다.

비유: 왜 프론트엔드와 백엔드를 나누나요?

마트에서 쇼핑할 때를 생각해보세요.
프론트엔드 = 매장: 예쁜 진열대, 가격표, 카트 → 손님이 보고 만지는 부분
백엔드 = 물류 창고: 재고 관리, 입출고, 회계 → 뒤에서 돌아가는 시스템

매장 디자인을 바꿔도 창고 시스템은 그대로이고, 창고 시스템을 업그레이드해도 매장은 그대로입니다. 이렇게 분리하면 독립적으로 관리할 수 있어서 효율적입니다.

실습

실습: 로컬에서 ShopEasy 실행하기
사전 준비물

아래 소프트웨어가 설치되어 있어야 합니다:
- Node.js (v18 이상): https://nodejs.org에서 LTS 버전 다운로드
- Git: https://git-scm.com에서 다운로드
- 설치 확인: 터미널에서 node --version 과 git --version 실행

  1. 프로젝트 코드 다운로드

    터미널을 열고 원하는 디렉토리에서 Git으로 코드를 클론합니다:

    bash 
    git clone https://github.com/your-org/ecommerce-app.git
cd ecommerce-app
    Git 사용이 어려운 경우

    강사가 USB 또는 메일로 전달한 ZIP 파일을 압축 해제하여 사용해도 됩니다.

  2. API 서버 설치 및 실행

    먼저 API 서버의 의존성을 설치합니다:

    bash 
    cd api-server
npm install

    테스트 데이터를 생성합니다 (상품, 리뷰 등의 샘플 데이터):

    bash 
    npm run seed

    API 서버를 시작합니다:

    bash 
    npm run dev

    정상적으로 실행되면 다음과 비슷한 메시지가 표시됩니다:

    text 
    Server is running on port 5000
Database connected successfully
  3. API 서버 동작 확인

    웹 브라우저를 열고 다음 URL에 접속합니다:

    text 
    http://localhost:5000/api/health

    {"status": "ok"}와 비슷한 응답이 나오면 API 서버가 정상입니다.

    상품 목록도 확인해봅시다:

    text 
    http://localhost:5000/api/products

    JSON 형태의 상품 데이터가 표시되면 성공입니다!

    [스크린샷: 브라우저에서 /api/products 응답 확인]
  4. 프론트엔드 설치 및 실행

    새로운 터미널 창을 열고 (API 서버는 계속 실행 상태 유지) 프론트엔드를 설치합니다:

    bash 
    cd ecommerce-app/frontend
npm install

    프론트엔드 개발 서버를 시작합니다:

    bash 
    npm run dev

    다음과 비슷한 메시지가 표시됩니다:

    text 
      VITE v5.x.x  ready in xxx ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: use --host to expose
  5. 쇼핑몰 확인

    웹 브라우저에서 http://localhost:5173에 접속합니다.

    ShopEasy 쇼핑몰 화면이 나타나면 성공입니다!

    [스크린샷: ShopEasy 메인 화면 - 상품 목록이 표시된 모습]

    다음을 확인해보세요:

    • 상품 목록이 정상적으로 표시되나요?
    • 상품을 클릭하면 상세 페이지로 이동하나요?
    • 장바구니에 상품을 담을 수 있나요?
    • 리뷰를 작성할 수 있나요?
  6. 서비스 종료

    실습이 끝나면 각 터미널에서 Ctrl + C를 눌러 서버를 종료합니다.

    현재 구조 정리

    현재 로컬에서 실행 중인 구조:
    - 프론트엔드: http://localhost:5173 (React 개발 서버)
    - API 서버: http://localhost:5000 (Node.js Express)
    - 데이터베이스: SQLite (api-server/data/ 폴더 안의 파일)

    다음 챕터부터 이 서비스를 AWS EC2에 올려보겠습니다!
