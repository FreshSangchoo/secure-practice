클라우드와 AWS 개요

학습 목표

클라우드 컴퓨팅이란?

클라우드 컴퓨팅이란, 내 컴퓨터가 아닌 인터넷을 통해 다른 곳에 있는 컴퓨터 자원(서버, 저장 공간, 데이터베이스 등)을 빌려서 사용하는 것입니다. 마치 전기를 직접 발전소를 세워서 만들 필요 없이, 전기회사에서 필요한 만큼 가져다 쓰는 것과 같습니다.

비유: 전기를 생각해보세요

1800년대에는 공장마다 자체 발전기를 설치했습니다. 비용도 많이 들고, 관리도 힘들었죠. 지금은 한전에서 전기를 보내주고, 우리는 콘센트에 꽂기만 하면 됩니다. 클라우드 컴퓨팅도 마찬가지입니다. 예전에는 회사마다 서버실을 직접 운영했지만, 이제는 AWS 같은 클라우드 업체가 서버를 관리해주고, 우리는 인터넷으로 접속해서 필요한 만큼 사용하면 됩니다.

클라우드의 5가지 핵심 특성

  • 온디맨드 셀프서비스: 필요할 때 사람에게 요청하지 않고 직접 클릭 몇 번으로 서버를 만들 수 있습니다
  • 광범위한 네트워크 접근: 인터넷만 되면 어디서든 접근할 수 있습니다
  • 자원 풀링: 여러 사용자가 같은 물리적 자원을 공유하지만, 각자 독립적으로 사용합니다
  • 빠른 탄력성: 사용량이 늘면 자원을 늘리고, 줄면 줄일 수 있습니다
  • 종량제 과금: 사용한 만큼만 비용을 지불합니다
비유: 수도 요금처럼

수도를 사용할 때 물탱크를 미리 채워놓지 않죠? 수도꼭지를 틀면 물이 나오고, 잠그면 멈춥니다. 요금도 쓴 만큼만 냅니다. 클라우드도 똑같습니다. 서버가 필요하면 켜고, 필요 없으면 끄면 됩니다. 켜놓은 시간만큼만 요금이 나옵니다.

피자로 이해하는 클라우드 서비스 모델

클라우드 서비스를 이해하는 가장 유명한 비유가 바로 "피자 비유"입니다. 피자를 먹는 방법에 따라 내가 직접 해야 할 일이 달라지듯, 클라우드도 서비스 모델에 따라 관리해야 할 범위가 달라집니다.

비유: 피자를 먹는 4가지 방법

1. 집에서 직접 만들어 먹기 (온프레미스)

반죽부터 토핑, 오븐까지 전부 직접 준비합니다. 가장 자유롭지만 가장 힘듭니다. 이것이 회사가 서버실을 직접 운영하는 것과 같습니다.

2. 밀키트로 만들어 먹기 (IaaS)

반죽과 기본 재료는 배달이 옵니다. 하지만 토핑을 얹고 굽는 건 내가 합니다. 서버(컴퓨터)는 빌리지만, 운영체제 설치와 소프트웨어 관리는 직접 합니다.

3. 배달 피자 (PaaS)

전화만 하면 완성된 피자가 옵니다. 그릇만 준비하면 되죠. 서버와 운영체제, 미들웨어까지 다 관리해주고, 나는 코드(애플리케이션)만 올리면 됩니다.

4. 피자 뷔페 (SaaS)

매장에 가서 앉기만 하면 됩니다. 아무것도 할 필요 없이 바로 먹습니다. Gmail, Notion 같은 서비스를 그냥 사용하는 것과 같습니다.

IaaS / PaaS / SaaS 비교

클라우드 서비스는 크게 세 가지 모델로 나뉩니다. 각 모델에 따라 클라우드 업체가 관리해주는 범위가 달라집니다.

구분 IaaS PaaS SaaS
뜻 Infrastructure as a Service Platform as a Service Software as a Service
한국어 서비스로서의 인프라 서비스로서의 플랫폼 서비스로서의 소프트웨어
사용자가 관리 OS, 미들웨어, 애플리케이션, 데이터 애플리케이션, 데이터 없음 (그냥 사용)
업체가 관리 하드웨어, 네트워크, 가상화 하드웨어부터 미들웨어까지 전부 전부
AWS 예시 EC2, VPC Elastic Beanstalk, Lambda WorkMail, Chime
일반 예시 가상 서버 임대 Heroku, Google App Engine Gmail, Dropbox, Notion
자유도 높음 중간 낮음
관리 부담 높음 중간 낮음
핵심 개념

이 강의에서는 주로 IaaS 수준의 서비스(EC2, VPC 등)를 다룹니다. 직접 서버를 관리하는 경험을 통해 클라우드의 기본을 확실히 이해한 뒤, 후반부에서 Lambda 같은 PaaS/서버리스 서비스로 넘어갑니다.

온프레미스 vs 클라우드

온프레미스(On-Premises)란 회사가 직접 서버실(데이터센터)을 구축하고 관리하는 전통적인 방식을 말합니다. "우리 건물 안(premises)에 서버가 있다"는 뜻이죠.

비유: 자가용 vs 택시

온프레미스 = 자가용: 내 차를 사면 항상 탈 수 있지만, 구매비, 보험, 정비, 주차비 등 유지비가 많이 듭니다. 사용 안 해도 감가상각은 됩니다.
클라우드 = 택시/카셰어링: 필요할 때만 부르면 됩니다. 초기 비용 없이 탄 만큼만 내면 됩니다. 대신 장기적으로 매일 타면 자가용보다 비쌀 수 있습니다.

비교 항목 온프레미스 클라우드
초기 비용 매우 높음 (서버 구매, 설치, 네트워크 구축) 거의 없음 (가입만 하면 시작)
운영 비용 고정비 (사용 안 해도 비용 발생) 변동비 (사용한 만큼 과금)
확장성 서버 추가 주문 → 배송 → 설치 (몇 주~몇 달) 클릭 몇 번으로 몇 분 안에 확장
글로벌 배포 해외 데이터센터 직접 구축 필요 원하는 리전 선택만 하면 됨
보안 직접 관리 (통제권 높음) 공동 책임 모델 (업체 + 사용자)
유지보수 전담 인력 필요 업체가 하드웨어 관리
실무 팁

대부분의 스타트업과 중소기업은 클라우드가 유리합니다. 대기업도 점점 클라우드로 전환(마이그레이션)하는 추세이며, 온프레미스와 클라우드를 함께 쓰는 하이브리드 클라우드 전략도 많이 사용됩니다.

AWS 역사와 시장 점유율

AWS(Amazon Web Services)는 세계 최대의 클라우드 서비스 제공업체입니다.

AWS의 간략한 역사

  • 2002년: Amazon이 내부 인프라를 서비스로 제공하는 아이디어 시작
  • 2006년: S3(스토리지)와 EC2(가상 서버) 출시 - 클라우드 시대의 시작
  • 2010년대: 폭발적 성장. Netflix, Airbnb 등 대형 서비스들이 AWS 위에서 운영
  • 현재: 200개 이상의 서비스, 전 세계 30개 이상의 리전 운영

클라우드 시장 점유율 (2024년 기준)

순위 업체 시장 점유율
1위 AWS (Amazon) 약 31%
2위 Azure (Microsoft) 약 25%
3위 GCP (Google) 약 11%
기타 Alibaba, Oracle, IBM 등 약 33%
왜 AWS를 배우나요?

AWS는 클라우드 시장 1위이자 가장 오래된 서비스입니다. 서비스 종류가 가장 많고, 관련 문서와 커뮤니티도 가장 활발합니다. AWS를 익히면 Azure나 GCP도 비슷한 개념이라 쉽게 적응할 수 있습니다.

AWS 글로벌 인프라

AWS는 전 세계에 데이터센터를 운영하고 있습니다. 이 인프라를 이해하는 것이 AWS 사용의 첫걸음입니다.

비유: 편의점 체인을 생각해보세요

GS25가 전국에 매장을 가지고 있죠? AWS도 전 세계에 데이터센터를 가지고 있습니다.
리전 = 도시 (서울, 부산, 도쿄...)
가용 영역 = 같은 도시 안의 여러 매장 (강남점, 홍대점...)
엣지 로케이션 = 택배 물류 거점 (빠른 배달을 위한 중간 보관소)

리전 (Region)

리전은 AWS 데이터센터가 모여 있는 지리적 영역입니다. 전 세계에 30개 이상의 리전이 있으며, 각 리전은 독립적으로 운영됩니다.

  • 서울 리전: ap-northeast-2 (아시아 태평양 - 서울)
  • 도쿄 리전: ap-northeast-1
  • 버지니아 리전: us-east-1 (AWS에서 가장 오래된 리전)
리전 선택 기준

리전을 선택할 때 고려할 점:
1. 지연 시간(Latency): 사용자와 가까운 리전을 선택 (한국 서비스 → 서울 리전)
2. 요금: 리전마다 요금이 다릅니다 (서울 > 버지니아)
3. 서비스 가용성: 모든 서비스가 모든 리전에 있는 건 아닙니다
4. 법적 규제: 데이터가 특정 국가 내에 있어야 하는 경우

가용 영역 (Availability Zone, AZ)

가용 영역은 하나의 리전 안에 있는 개별 데이터센터(또는 데이터센터 그룹)입니다. 서울 리전에는 4개의 가용 영역이 있습니다 (ap-northeast-2a, 2b, 2c, 2d).

각 가용 영역은 물리적으로 떨어져 있어서, 하나가 자연재해로 다운되어도 다른 가용 영역은 정상 운영됩니다.

비유: 달걀을 한 바구니에 담지 마세요

중요한 서비스는 여러 가용 영역에 분산 배치합니다. 마치 중요한 문서를 USB, 외장하드, 클라우드 세 곳에 백업하는 것처럼요. 하나가 고장 나도 다른 곳에서 서비스를 계속할 수 있습니다.

엣지 로케이션 (Edge Location)

엣지 로케이션은 콘텐츠를 사용자에게 빠르게 전달하기 위한 캐시 서버입니다. 전 세계 400개 이상의 지점에 배치되어 있으며, CloudFront(CDN) 서비스가 이를 활용합니다.

비유: 쿠팡 물류 센터

쿠팡이 전국에 물류 센터를 두는 이유는 빠른 배송을 위해서입니다. 엣지 로케이션도 마찬가지입니다. 미국에 있는 원본 데이터를 매번 가져오면 느리니까, 한국의 엣지 로케이션에 캐시(복사본)를 저장해두고 빠르게 전달합니다.

AWS 프리티어 소개

AWS는 처음 사용하는 분들을 위해 프리티어(Free Tier)를 제공합니다. 일정 범위 내에서 무료로 AWS 서비스를 사용할 수 있어, 학습이나 테스트에 매우 유용합니다.

프리티어의 3가지 유형

유형 설명 대표 서비스
12개월 무료 AWS 계정 생성 후 12개월 동안 무료 EC2 t2.micro (월 750시간), S3 (5GB), RDS (월 750시간)
항상 무료 12개월 제한 없이 항상 무료 Lambda (월 100만 요청), DynamoDB (25GB), SNS (월 100만 게시)
단기 체험 특정 서비스 단기 체험 SageMaker, Redshift 등 (서비스마다 다름)
비용 주의!

프리티어라고 해서 모든 것이 무료는 아닙니다! 꼭 주의하세요:
1) 프리티어 범위를 초과하면 요금이 청구됩니다 (예: t2.micro가 아닌 t2.small을 선택하면 과금)
2) 12개월 무료 서비스는 계정 생성 12개월 후 자동으로 과금이 시작됩니다
3) 리전을 잘못 선택하면 의도치 않은 리소스가 다른 리전에 생길 수 있습니다
4) 실습 후 반드시 리소스를 정리(삭제)하세요!

프리티어 범위 확인하기

AWS 콘솔에서 Billing & Cost Management → Free Tier에서 현재 프리티어 사용량을 확인할 수 있습니다. 이 강의에서는 Chapter 07에서 비용 관리 방법을 자세히 다룹니다.

비유: 쇼핑몰 건물을 짓는 과정

우리가 배울 과정을 건물 짓기에 비유하면:
1단계: 땅 확보 (AWS 계정 생성, IAM)
2단계: 기초 공사 (VPC - 네트워크 설계)
3단계: 건물 올리기 (EC2 - 서버 배포)

7단계: 보안 시스템 (보안그룹, NACL)
8단계: CCTV 설치 (CloudWatch - 모니터링)
