EC2란?

EC2(Elastic Compute Cloud)는 AWS에서 가장 기본이 되는 서비스입니다. 한마디로 클라우드에 있는 가상 컴퓨터(서버)입니다. 물리적인 서버를 구매하지 않고도, 몇 분 만에 원하는 사양의 서버를 만들고 사용할 수 있습니다.

비유: 클라우드에 있는 내 컴퓨터

집에 데스크톱 PC를 한 대 사면 CPU, 메모리, 하드디스크가 정해져 있죠? EC2도 마찬가지입니다. CPU, 메모리, 저장 공간을 선택해서 나만의 서버를 만듭니다. 다만 이 서버는 내 책상 위가 아니라 AWS 데이터센터에 있고, 인터넷으로 원격 접속해서 사용합니다.

또한 PC를 산 뒤에 메모리를 추가하려면 부품을 사서 끼워야 하지만, EC2는 클릭 몇 번으로 사양을 바꿀 수 있습니다. 이것이 "Elastic(탄력적)"의 의미입니다.

EC2 용어 정리

  • 인스턴스(Instance): 실행 중인 가상 서버 하나를 인스턴스라고 부릅니다
  • 인스턴스 시작(Launch): 새 서버를 만들어서 실행하는 것
  • 인스턴스 중지(Stop): 서버를 끄는 것 (데이터는 유지, 요금 감소)
  • 인스턴스 종료(Terminate): 서버를 완전히 삭제하는 것 (데이터도 삭제)
중지 vs 종료 주의!

중지(Stop)는 컴퓨터 전원을 끄는 것과 같습니다. 나중에 다시 켤 수 있습니다.
종료(Terminate)는 컴퓨터를 폐기하는 것과 같습니다. 되돌릴 수 없습니다!
실습 중에 "종료" 버튼을 실수로 누르지 않도록 주의하세요.

인스턴스 타입

인스턴스 타입은 서버의 사양(CPU, 메모리, 네트워크 성능)을 결정합니다. 용도에 따라 다양한 타입이 있습니다.

비유: 자동차 등급

경차(t2.micro) → 출퇴근용, 저렴, 기본 성능
세단(m5.large) → 일반 업무용, 균형 잡힌 성능
SUV(r5.xlarge) → 많은 짐(메모리)을 실어야 할 때
스포츠카(c5.2xlarge) → 빠른 연산(CPU)이 필요할 때
트럭(i3.xlarge) → 대용량 저장이 필요할 때

인스턴스 타입 이름 읽는 법

t2.micro를 예시로 분석해봅시다:

  • t = 인스턴스 패밀리 (t: 범용 버스터블, m: 범용, c: 컴퓨팅 최적화, r: 메모리 최적화)
  • 2 = 세대 (숫자가 클수록 최신 세대, 성능 개선)
  • micro = 크기 (nano < micro < small < medium < large < xlarge < 2xlarge ...)

주요 인스턴스 타입

타입 vCPU 메모리 프리티어 용도
t2.micro 1 1 GB O (월 750시간) 학습, 테스트, 소규모 웹
t3.micro 2 1 GB O (일부 리전) 경량 워크로드
t3.small 2 2 GB X 소규모 애플리케이션
t3.medium 2 4 GB X 중소규모 애플리케이션
m5.large 2 8 GB X 범용 프로덕션
프리티어 주의!

이 강의에서는 항상 t2.micro를 사용합니다. 다른 인스턴스 타입을 선택하면 요금이 발생합니다! 인스턴스 생성 시 반드시 "프리 티어 사용 가능" 라벨이 있는지 확인하세요.

AMI란?

AMI(Amazon Machine Image)는 EC2 인스턴스를 만들 때 사용하는 운영체제 + 소프트웨어가 포함된 템플릿입니다.

비유: 스마트폰 공장 초기화 이미지

새 스마트폰을 사면 안드로이드(또는 iOS)가 이미 설치되어 있죠? AMI도 마찬가지입니다. EC2 인스턴스를 만들 때 어떤 운영체제를 사용할지 선택합니다. 리눅스(Amazon Linux, Ubuntu)를 설치할 수도 있고, Windows를 설치할 수도 있습니다.

자주 사용하는 AMI

AMI 설명 이 강의에서
Amazon Linux 2023 AWS가 만든 리눅스. AWS 서비스와 궁합이 좋음 사용
Ubuntu 가장 인기 있는 리눅스 배포판 -
Windows Server Windows 기반 서버 -

키페어 (Key Pair)

키페어는 EC2 인스턴스에 SSH(원격 접속)로 로그인하기 위한 보안 열쇠입니다. 비밀번호 대신 키 파일(.pem)을 사용합니다.

비유: 디지털 도어록의 열쇠

아파트 디지털 도어록에는 비밀번호도 있고, 물리적 열쇠도 있죠? EC2의 키페어는 이 물리적 열쇠와 같습니다.

공개 키(Public Key): 도어록에 등록된 잠금 장치 (EC2에 저장됨)
개인 키(Private Key): 내가 가지고 있는 열쇠 (.pem 파일)

열쇠(개인 키)를 가진 사람만 문(EC2)을 열 수 있습니다. 열쇠를 잃어버리면 문을 열 수 없으니 반드시 안전한 곳에 보관하세요!

키페어 분실 주의!

키페어 파일(.pem)은 생성할 때 딱 한 번만 다운로드할 수 있습니다. 분실하면 해당 EC2에 SSH로 접속할 수 없습니다. 다운로드 후 안전한 곳에 보관하고, 절대 다른 사람에게 공유하지 마세요.

보안그룹 (Security Group)

보안그룹은 EC2 인스턴스의 가상 방화벽입니다. 어떤 트래픽(접속)을 허용하고, 어떤 트래픽을 차단할지 규칙을 설정합니다.

비유: 집의 도어락 + 인터폰

집에 아무나 들어올 수 없죠? 인터폰으로 확인하고, 허용된 사람만 문을 열어줍니다. 보안그룹도 마찬가지입니다.

- "SSH(22번 포트)는 내 IP에서만 접속 허용" = "내 가족만 출입 가능"
- "HTTP(80번 포트)는 모든 곳에서 허용" = "택배 기사는 현관까지 출입 가능"
- "3306번 포트는 차단" = "낯선 사람 출입 금지"

보안그룹 규칙의 구성 요소

요소 설명 예시
유형(Type) 프로토콜 종류 SSH, HTTP, HTTPS, Custom TCP
포트(Port) 통신에 사용할 포트 번호 22 (SSH), 80 (HTTP), 5000 (API)
소스(Source) 접속을 허용할 IP 범위 0.0.0.0/0 (모든 곳), 내 IP
인바운드 vs 아웃바운드

인바운드(Inbound): 외부에서 EC2로 들어오는 트래픽 규칙 (기본: 모두 차단)
아웃바운드(Outbound): EC2에서 외부로 나가는 트래픽 규칙 (기본: 모두 허용)

웹 서버를 운영하려면 인바운드 규칙에 HTTP(80) 포트를 열어야 합니다.

실습

실습: EC2에 API 서버 배포하기
  1. EC2 대시보드 이동

    AWS 콘솔에 IAM 사용자로 로그인합니다.

    리전이 서울(ap-northeast-2)인지 확인합니다.

    상단 검색바에 "EC2"를 입력하고 클릭합니다.

    왼쪽 메뉴에서 "인스턴스(Instances)"를 클릭합니다.

    [스크린샷: EC2 대시보드 - 인스턴스 메뉴]
  2. 인스턴스 시작하기

    오른쪽 상단의 "인스턴스 시작(Launch instances)" 버튼을 클릭합니다.

  3. 이름 및 태그 설정

    Name: ShopEasy-API를 입력합니다.

    이 이름은 나중에 여러 인스턴스를 구분하는 데 사용됩니다.

  4. AMI 선택

    "Amazon Linux 2023 AMI"를 선택합니다.

    "프리 티어 사용 가능(Free tier eligible)" 라벨이 있는지 확인하세요.

    Architecture는 "64-bit (x86)"을 선택합니다.

    [스크린샷: AMI 선택 화면 - Amazon Linux 2023 선택]
  5. 인스턴스 타입 선택

    t2.micro를 선택합니다. (기본으로 선택되어 있을 수 있습니다)

    "프리 티어 사용 가능" 라벨을 반드시 확인하세요!

    반드시 t2.micro!

    다른 인스턴스 타입을 선택하면 시간당 요금이 발생합니다. t2.micro만 프리티어 무료입니다.

  6. 키페어 생성

    "새 키 페어 생성(Create new key pair)"을 클릭합니다.

    • Key pair name: shopeasy-key
    • Key pair type: RSA
    • Private key file format: .pem (macOS/Linux) 또는 .ppk (Windows PuTTY 사용 시)

    "Create key pair"를 클릭하면 키 파일이 자동으로 다운로드됩니다.

    키 파일을 반드시 안전한 곳에 저장!

    다운로드된 shopeasy-key.pem 파일은 이때만 다운로드할 수 있습니다. 분실하면 EC2에 SSH 접속이 불가능합니다. 바탕화면이 아닌 안전한 폴더에 저장하세요.

    [스크린샷: 키페어 생성 팝업]
  7. 네트워크 설정 (보안그룹)

    "네트워크 설정(Network settings)" 옆의 "편집(Edit)"을 클릭합니다.

    보안그룹 이름: ShopEasy-SG

    설명: ShopEasy security group

    다음 인바운드 규칙을 추가합니다:

    유형 포트 소스 설명
    SSH 22 내 IP (My IP) SSH 원격 접속용
    HTTP 80 0.0.0.0/0 (Anywhere) 웹 서비스 접속용
    사용자 지정 TCP 5000 0.0.0.0/0 (Anywhere) API 서버 접속용

    "보안 그룹 규칙 추가(Add security group rule)" 버튼을 눌러 규칙을 추가합니다.

    [스크린샷: 보안그룹 규칙 설정 화면]
  8. 스토리지 설정

    기본값 8 GiB gp3를 그대로 사용합니다. (프리티어 범위: 30GB까지)

  9. 인스턴스 시작!

    오른쪽의 "인스턴스 시작(Launch instance)" 버튼을 클릭합니다.

    "성공적으로 시작되었습니다(Successfully initiated launch)" 메시지가 나오면 성공!

    "모든 인스턴스 보기(View all instances)"를 클릭합니다.

    인스턴스 상태가 "실행 중(Running)"이 될 때까지 약 1~2분 기다립니다.

    [스크린샷: 인스턴스 목록 - Running 상태 확인]
  10. 퍼블릭 IP 확인

    인스턴스를 선택하고 아래 "세부 정보(Details)" 탭에서 "퍼블릭 IPv4 주소(Public IPv4 address)"를 찾아 메모합니다.

    예시: 3.35.xxx.xxx

    퍼블릭 IP란?

    인터넷에서 이 EC2에 접속하기 위한 주소입니다. 이 IP로 SSH 접속을 하고, 나중에 웹 브라우저로도 접속합니다.

  11. SSH로 EC2 접속

    macOS/Linux: 터미널을 열고 다음 명령어를 실행합니다:

    bash 
    # 키 파일 권한 변경 (최초 1회)
chmod 400 ~/Downloads/shopeasy-key.pem

# SSH 접속
ssh -i ~/Downloads/shopeasy-key.pem ec2-user@[EC2-퍼블릭-IP]

    Windows (PowerShell):

    powershell 
    # SSH 접속
ssh -i C:\Users\사용자명\Downloads\shopeasy-key.pem ec2-user@[EC2-퍼블릭-IP]

    "Are you sure you want to continue connecting?" 질문에 yes를 입력합니다.

    프롬프트가 [ec2-user@ip-xxx-xxx-xxx-xxx ~]$ 형태로 바뀌면 접속 성공!

    [스크린샷: SSH 접속 성공 화면]
    Windows에서 SSH가 안 되는 경우

    Windows 10 이전 버전에서는 SSH 클라이언트가 기본 설치되어 있지 않을 수 있습니다. 이 경우 PuTTY를 사용하세요. 키 파일(.pem)을 PuTTYgen으로 .ppk 형식으로 변환 후 사용합니다. 또는 AWS 콘솔의 "EC2 Instance Connect" 기능으로 브라우저에서 직접 접속할 수도 있습니다.

  12. Node.js 설치

    EC2에 접속한 상태에서 Node.js를 설치합니다:

    bash 
    # 시스템 업데이트
sudo dnf update -y

# Node.js 20.x 설치 (Amazon Linux 2023)
sudo dnf install -y nodejs20

# 설치 확인
node --version
npm --version

    node --version이 v20.x.x로 나오면 성공입니다.

  13. Git 설치 및 프로젝트 코드 다운로드
    bash 
    # Git 설치
sudo dnf install -y git

# 프로젝트 클론
git clone https://github.com/your-org/ecommerce-app.git
cd ecommerce-app/api-server
    Git 대신 scp로 업로드하는 방법

    GitHub에 코드가 없는 경우, 로컬에서 직접 파일을 업로드할 수 있습니다.
    로컬 터미널에서 (EC2가 아닌 내 컴퓨터에서):

    bash 
    # 로컬 터미널에서 실행 (EC2가 아님!)
scp -i ~/Downloads/shopeasy-key.pem -r ./api-server ec2-user@[EC2-IP]:~/ecommerce-app/api-server
  14. 의존성 설치 및 데이터 시딩
    bash 
    # api-server 디렉토리에서
npm install

# 테스트 데이터 생성
npm run seed
  15. API 서버 실행
    bash 
    # API 서버 시작
npm start

    서버가 시작되었다는 메시지가 나오면 성공입니다.

    백그라운드 실행 (선택사항)

    SSH 연결을 끊어도 서버가 계속 실행되게 하려면 nohup을 사용합니다:

    bash 
    # 백그라운드 실행
nohup npm start > server.log 2>&1 &

# 로그 확인
tail -f server.log

# 프로세스 확인
ps aux | grep node
  16. 브라우저에서 API 확인

    내 컴퓨터의 웹 브라우저에서 다음 URL로 접속합니다:

    text 
    http://[EC2-퍼블릭-IP]:5000/api/health

    {"status": "ok"} 응답이 나오면 EC2에 API 서버 배포 성공!

    상품 목록도 확인합니다:

    text 
    http://[EC2-퍼블릭-IP]:5000/api/products
    [스크린샷: 브라우저에서 EC2 IP로 API 응답 확인]
여기까지의 상태 정리

지금까지 완성한 구조:

사용자 브라우저 → (인터넷) → EC2 (API 서버, 포트 5000) → SQLite DB
