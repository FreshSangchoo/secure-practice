Chapter 14

API 서버 → RDS 연동

학습 목표

  • API 서버의 데이터베이스 연결을 SQLite에서 RDS MySQL로 전환한다
  • 환경변수를 통한 데이터베이스 설정 변경 방법을 이해한다
  • RDS에 시드 데이터를 생성하여 서비스를 복원한다
  • 브라우저에서 RDS 연동이 정상적인지 확인한다

현재 상태

현재 ShopEasy API 서버는 SQLite를 사용하고 있습니다. SQLite는 파일 기반 DB로 별도의 DB 서버가 필요 없어 간편하지만, 실서비스에는 부적합합니다.

비유로 이해하기: 개인 수첩 vs 공유 장부

현재는 각 직원(API 서버)이 개인 수첩(SQLite)에 주문을 기록하는 상태입니다. 직원이 퇴사(EC2 종료)하면 수첩도 없어지고, 직원마다 기록이 다릅니다. 우리가 해야 할 일은 공유 장부(RDS)를 사무실 금고에 두고, 모든 직원이 같은 장부를 보는 것입니다.

핵심 개념: 환경변수로 DB 전환

ShopEasy API 서버는 환경변수에 따라 SQLite와 MySQL을 자동으로 전환하도록 설계되어 있습니다. .env 파일에서 DB_TYPE=mysql로 변경하고 RDS 접속 정보를 넣으면, 코드 수정 없이 DB가 전환됩니다.

환경변수 변경 방법

ShopEasy API 서버의 .env 파일에서 데이터베이스 관련 환경변수를 변경합니다.

SQLite 설정 (현재)

env
PORT=5000
DB_TYPE=sqlite
NODE_ENV=production

MySQL (RDS) 설정 (변경 후)

env
PORT=5000
DB_TYPE=mysql
DB_HOST=shopeasy-db.xxxxx.ap-northeast-2.rds.amazonaws.com
DB_PORT=3306
DB_NAME=shopeasy
DB_USER=admin
DB_PASSWORD=YourPassword123!
NODE_ENV=production
비유로 이해하기: 네비게이션 목적지 변경

환경변수를 변경하는 것은 네비게이션의 목적지를 변경하는 것과 같습니다. 차(API 서버)는 그대로인데, 목적지(DB 주소)만 바꾸면 다른 곳으로 갑니다. "동네 편의점(SQLite)"에서 "대형마트(RDS MySQL)"로 목적지만 바꾸면 됩니다!

환경변수 설명

변수설명예시 값
DB_TYPE사용할 DB 종류mysql
DB_HOSTRDS 엔드포인트shopeasy-db.xxxxx.rds.amazonaws.com
DB_PORTMySQL 포트3306
DB_NAME데이터베이스 이름shopeasy
DB_USERDB 사용자 이름admin
DB_PASSWORDDB 비밀번호YourPassword123!
보안 주의: 비밀번호 관리

실습에서는 .env 파일에 비밀번호를 직접 넣지만, 실무에서는 AWS Secrets Manager나 Parameter Store를 사용하여 비밀번호를 안전하게 관리합니다. .env 파일이 Git에 커밋되지 않도록 .gitignore에 반드시 추가하세요.

실습

실습: ShopEasy API 서버를 RDS MySQL에 연동하기
사전 준비

Chapter 13의 RDS MySQL 인스턴스가 Available 상태여야 합니다. RDS 엔드포인트와 마스터 비밀번호를 확인하세요.

  1. API 서버 EC2에 SSH 접속

    Bastion Host(프론트엔드 EC2)를 경유하여 API 서버 EC2에 접속합니다.

    bash
    # 프론트엔드 EC2 접속
ssh -A -i my-keypair.pem ec2-user@<프론트엔드-퍼블릭-IP>

# API 서버 EC2로 이동
ssh ec2-user@<API-서버-프라이빗-IP>
  2. MySQL 클라이언트 설치 및 RDS 접속 확인
    bash
    # MySQL 클라이언트 설치 (이전 챕터에서 이미 했다면 건너뛰기)
sudo dnf install -y mariadb105

# RDS 접속 테스트
mysql -h shopeasy-db.xxxxx.ap-northeast-2.rds.amazonaws.com -u admin -p

    접속 후 데이터베이스를 확인합니다:

    sql
    SHOW DATABASES;
-- shopeasy 데이터베이스가 보이면 정상

USE shopeasy;
SHOW TABLES;
-- 아직 테이블이 없는 것이 정상 (seed로 생성 예정)

EXIT;
    [스크린샷: MySQL 클라이언트로 RDS 접속 성공]
  3. .env 파일 수정 (DB 설정 변경)

    API 서버의 환경변수를 SQLite에서 MySQL로 변경합니다.

    bash
    # API 서버 디렉토리로 이동
cd ~/ecommerce-app/api-server

# 기존 .env 백업
cp .env .env.backup

# .env 파일 수정
cat > .env <<'EOF'
PORT=5000
DB_TYPE=mysql
DB_HOST=shopeasy-db.xxxxx.ap-northeast-2.rds.amazonaws.com
DB_PORT=3306
DB_NAME=shopeasy
DB_USER=admin
DB_PASSWORD=YourPassword123!
NODE_ENV=production
EOF
    RDS 엔드포인트를 정확히 입력하세요!

    DB_HOST에는 Chapter 13에서 확인한 RDS 엔드포인트를 입력합니다. xxxxx 부분을 실제 값으로 변경하세요. AWS 콘솔 → RDS → shopeasy-db → Connectivity & security에서 확인할 수 있습니다.

    .env 파일이 올바르게 설정되었는지 확인:

    bash
    # .env 내용 확인
cat .env
    [스크린샷: .env 파일 내용 확인]
  4. MySQL 드라이버 설치

    Node.js에서 MySQL에 접속하기 위한 드라이버를 설치합니다.

    bash
    # mysql2 패키지 설치 (Node.js MySQL 드라이버)
cd ~/ecommerce-app/api-server
npm install mysql2
    mysql2 패키지란?

    mysql2는 Node.js에서 MySQL/MariaDB에 접속하기 위한 가장 인기 있는 드라이버입니다. ShopEasy API 서버 코드에 이미 mysql2를 지원하는 로직이 포함되어 있습니다.

  5. 시드 데이터 생성 (RDS에 초기 데이터 입력)

    RDS에 테이블을 생성하고 샘플 상품 데이터를 넣는 시드 명령을 실행합니다.

    bash
    # 시드 데이터 생성
cd ~/ecommerce-app/api-server
npm run seed

    정상적으로 실행되면 다음과 비슷한 출력이 나옵니다:

    bash
    # 예상 출력:
# Connected to MySQL database
# Creating tables...
# Tables created successfully
# Seeding products...
# 10 products inserted
# Seeding complete!
    seed 실행 시 오류가 나면?

    흔한 오류와 해결:

    • ECONNREFUSED: RDS 엔드포인트 확인, 보안그룹 확인
    • Access denied: DB_USER, DB_PASSWORD 확인
    • Unknown database: RDS 생성 시 Initial database name을 shopeasy로 설정했는지 확인

    MySQL 클라이언트로 데이터가 생성되었는지 확인합니다:

    bash
    # RDS 접속
mysql -h shopeasy-db.xxxxx.ap-northeast-2.rds.amazonaws.com -u admin -p
    sql
    USE shopeasy;

-- 테이블 확인
SHOW TABLES;

-- 상품 데이터 확인
SELECT id, name, price FROM products LIMIT 5;

-- 결과 예시:
-- +----+------------------+-------+
-- | id | name             | price |
-- +----+------------------+-------+
-- |  1 | 무선 블루투스 이어폰 | 45000 |
-- |  2 | 스마트 워치       | 89000 |
-- |  3 | USB-C 허브        | 35000 |
-- ...

EXIT;
    [스크린샷: RDS에서 상품 데이터 확인]
  6. API 서버 재시작

    환경변수를 변경했으므로 API 서버를 재시작합니다.

    bash
    # PM2로 관리 중인 경우
pm2 restart shopeasy-api

# 상태 확인
pm2 status

# 로그 확인 (MySQL 연결 성공 메시지 확인)
pm2 logs shopeasy-api --lines 20

    로그에서 MySQL 연결 성공 메시지를 확인합니다:

    bash
    # 로그에 다음과 같은 메시지가 보이면 성공:
# Server running on port 5000
# Connected to MySQL database at shopeasy-db.xxxxx.rds.amazonaws.com
    로그에서 DB 연결 확인이 중요합니다

    서버가 시작되었더라도 DB 연결에 실패하면 API 요청 시 오류가 발생합니다. 반드시 로그에서 "Connected to MySQL" 메시지를 확인하세요.

  7. API 응답 확인

    API 서버가 RDS에서 데이터를 정상적으로 가져오는지 확인합니다.

    bash
    # API 서버 로컬에서 테스트
curl http://localhost:5000/api/products

# 상품 목록 JSON이 반환되면 성공!

    ALB를 통한 접근도 확인합니다:

    bash
    # ALB DNS로 접근 테스트 (프론트엔드 EC2 또는 로컬에서)
curl http://alb-shopeasy-123456789.ap-northeast-2.elb.amazonaws.com/api/products
    [스크린샷: API 응답 - RDS에서 가져온 상품 데이터]
  8. 브라우저에서 ShopEasy 확인

    브라우저에서 프론트엔드에 접속하여 상품 목록이 정상적으로 표시되는지 확인합니다.

    text
    http://<프론트엔드-퍼블릭-IP>
    • 상품 목록이 정상적으로 로드되는지 확인
    • 상품 상세 페이지가 열리는지 확인
    • 장바구니 기능이 동작하는지 확인
    [스크린샷: 브라우저에서 ShopEasy - 상품 목록 표시]
    비유로 이해하기: 성공적인 이전!

    우리는 방금 식당의 개인 수첩(SQLite)을 공유 장부(RDS)로 교체했습니다. 식당 운영(쇼핑몰 서비스)은 고객에게 아무런 차이 없이 계속 됩니다. 하지만 이제부터 직원(API 서버)이 몇 명이 되든 모두 같은 장부를 보고, 장부는 금고(프라이빗 서브넷)에 안전하게 보관됩니다!

  9. (추가) Auto Scaling 환경에서의 .env 관리

    Auto Scaling으로 새 EC2가 생성될 때도 RDS 연결 설정이 적용되어야 합니다. 시작 템플릿의 User Data를 업데이트합니다.

    AWS 콘솔 → EC2 → Launch Templates → lt-shopeasy-api → Modify template (Create new version)

    Advanced details → User data를 다음으로 변경:

    bash
    #!/bin/bash
# .env 파일 설정
cat > /home/ec2-user/ecommerce-app/api-server/.env <<'ENVEOF'
PORT=5000
DB_TYPE=mysql
DB_HOST=shopeasy-db.xxxxx.ap-northeast-2.rds.amazonaws.com
DB_PORT=3306
DB_NAME=shopeasy
DB_USER=admin
DB_PASSWORD=YourPassword123!
NODE_ENV=production
ENVEOF

# API 서버 시작
cd /home/ec2-user/ecommerce-app/api-server
sudo -u ec2-user pm2 start npm --name "shopeasy-api" -- start
sudo -u ec2-user pm2 save

    시작 템플릿의 기본 버전을 새 버전으로 변경:

    Launch Templates → lt-shopeasy-api → Actions → Set default version

    User Data에 비밀번호가 들어있습니다

    학습 환경에서는 괜찮지만, 실무에서는 AWS Secrets Manager나 Systems Manager Parameter Store에서 비밀번호를 가져오도록 User Data를 작성합니다.

    AMI도 새로 생성해야 합니다 (mysql2 패키지가 설치된 상태로):

    bash
    # API 서버 EC2에서 새 AMI 생성
# AWS 콘솔 > EC2 > Instances > API 서버 선택 > Actions > Image > Create image
# Image name: shopeasy-api-v2-mysql
이번 챕터에서 달성한 것
  • API 서버의 데이터베이스를 SQLite에서 RDS MySQL로 전환
  • 환경변수(.env) 변경만으로 DB 전환 완료 (코드 수정 없음)
  • RDS에 시드 데이터를 생성하여 상품 정보 이전
  • 브라우저에서 RDS 연동 정상 동작 확인
  • Auto Scaling 시작 템플릿에 RDS 설정 반영
